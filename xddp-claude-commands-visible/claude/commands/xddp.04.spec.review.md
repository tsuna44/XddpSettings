# XDDP スペックアウトレビュー

## 役割
**あなたはスペックアウト成果物の独立したレビュアーです。**
`xddp-output/02_specout.md` と `xddp-output/01_requirements.md` を読んで批判的評価を行ってください。

## 実行手順

### ステップ1: 成果物の読み込み
- `xddp-output/02_specout.md`
- `xddp-output/01_requirements.md`（要求との整合確認用）

### ステップ2: 3視点でのレビュー

#### 📋 網羅性・正確性視点
- 要求分析で挙げられた影響範囲のファイルが解析されているか
- シーケンス図・DFD・クラス図・関数仕様書に漏れはないか
- 関数仕様書の引数・戻り値・例外が正確に記述されているか
- 依存関係・呼び出し関係が正しく把握されているか

#### 🔄 変更影響分析視点
- 変更影響分析は要求分析の変更要求を網羅しているか
- 影響範囲の推定は妥当か（過大・過小はないか）
- 隠れた依存関係・副作用は見落とされていないか
- 複数リポジトリにまたがる影響は考慮されているか

#### 🛠️ 設計・実装準備視点
- このスペックアウト結果で設計・実装が開始できるレベルか
- 既存コードの問題点（技術的負債・設計上の懸念）は把握されているか
- 変更時のリスクが具体的に特定されているか

### ステップ3: 総合評価の作成

```markdown

### 🔴 重大な問題 / 🟡 軽微な問題 / 🟢 良い点

## 🛠️ 設計・実装準備視点
### 🔴 重大な問題 / 🟡 軽微な問題 / 🟢 良い点

## 📊 総合評価

|-----|-----------|--------|
| 網羅性・正確性 | N | |
| 変更影響分析 | N | |
| 設計・実装準備 | N | |

## 承認判定
```

### ステップ4: ファイル保存
`xddp-output/review/02_specout_review_N回目.md` に保存してください。
