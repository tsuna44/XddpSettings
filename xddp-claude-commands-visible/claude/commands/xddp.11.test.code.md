# XDDP テストコード実装

## 役割
**承認済みのテスト仕様書**を元にテストコードを実装してください。
このコマンドは `/xddp.09.test.spec` → `/xddp.10.test.review` → 人による承認の後に実行します。

## 事前確認
- `xddp-output/05_testing_spec.md` が存在するか
- `xddp-output/review/05_testing_spec_review_*` が存在するか（レビュー済みか）

どちらかが存在しない場合は先に該当コマンドを実行するよう案内してください。

## 実行手順

### ステップ1: 承認済みテスト仕様書の読み込み
`xddp-output/05_testing_spec.md` の全テストケースを把握してください。

### ステップ2: 既存テストコードの確認
テストフレームワーク・ファイル命名規則・フィクスチャ・モックのスタイルを把握してください。

### ステップ3: テストコードの実装
仕様書の全テストケースを網羅するテストコードを `tests/` フォルダに作成してください。
- TC-IDをテスト関数名・コメントに明記する（例：`def test_TC001_notify_slack_success():` ）
- 日本語コメントでテスト意図を記述する
- 正常系・異常系・境界値をすべて実装する
- モック・フィクスチャは既存スタイルに合わせる

### ステップ4: テストの構文確認
```bash
python -m pytest tests/ --collect-only  # Python
npx jest --listTests                    # Node.js
```
エラーがあれば修正してください。

### ステップ5: 成果物ドキュメントの作成と保存

```markdown
# XDDP テストコード実装完了

**生成日時:** YYYY-MM-DD HH:MM

## 実装サマリー
| テストファイル数 | テストケース数（仕様/実装） | 未実装 |

## 作成したテストファイル一覧
| ファイルパス | テスト対象 | TC-ID範囲 |

## 仕様書との対応確認
| TC-ID | テスト関数名 | 実装ファイル | ステータス |

## テスト実行方法
```

`xddp-output/06_testing_code.md` と `xddp-output/XDDP_summary.md` を保存してください。

## 完了後の案内
```
✅ テストコード実装完了

💾 テストコード: tests/ フォルダに保存済み

次のステップ：
  /xddp.08.code.review ← テストコードのレビューも推奨
  /xddp.status      ← 全フェーズサマリーを確認
```
