# XDDP コーディングレビュー

## 役割
**あなたはコーディング成果物の独立したレビュアーです。**
設計書・実装ファイルを読んで批判的評価を行ってください。

## 実行手順

### ステップ1: 成果物の読み込み
- `xddp-output/04_coding.md`（コーディング成果物）
- `xddp-output/03_design.md`（設計書との整合確認）
- 実際に作成・変更されたソースファイルを読み込む

### ステップ2: 3視点でのレビュー

#### 💎 コード品質・保守性視点
- 設計書との整合性はあるか（設計から逸脱していないか）
- 既存コードのスタイル・命名規則に合っているか
- 可読性・保守性は十分か（複雑すぎる処理はないか）
- 重複コード・不要なコードはないか
- 日本語コメントは適切に付与されているか

#### 🔐 セキュリティ・堅牢性視点
- 入力値バリデーションは適切か
- 認証・認可の処理に漏れはないか
- エラーハンドリングは網羅されているか
- 例外処理・ログ出力は適切か
- 機密情報（APIキー等）がハードコードされていないか

#### ⚡ パフォーマンス・信頼性視点
- パフォーマンス上の問題（N+1クエリ・無駄なループ等）はないか
- 並行処理・競合状態の考慮はあるか
- リソース（接続・ファイル等）のリークはないか
- 既存機能への回帰リスクはないか

### ステップ3: 総合評価の作成

## ⚡ パフォーマンス・信頼性視点

## 📊 総合評価

## 承認判定
- [ ] 🔴 重大問題あり → `/xddp.revise` を実行

### ステップ4: ファイル保存
`xddp-output/review/04_coding_review_N回目.md` に保存してください。
