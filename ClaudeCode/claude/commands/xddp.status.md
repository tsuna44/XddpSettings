---
description: XDDP 進捗確認: CRの現在フェーズと成果物の状態を確認する
---

指定されたCRの進捗状況を確認し、現在のフェーズと次のアクションを出力する。

## 実行内容

1. CR番号（引数）に対応する成果物を収集する：
   - `変更要求仕様書-CRS-{CR番号}.md`
   - `スペックアウト-{CR番号}.md`
   - `変更設計書-{CR番号}.md`
   - `テストケース-{CR番号}.md` または `.xlsx`
   - `テスト結果記録-{CR番号}.md`
   - `修正ループ管理-{CR番号}.md`
2. 各成果物の存在・ステータスを確認する
3. 現在のフェーズを判定する

## 出力形式
```
## XDDP 進捗状況

**CR番号**：CR-YYYY-NNN
**確認日時**：YYYY-MM-DD

### フェーズ別ステータス
| フェーズ | 成果物 | 状態 | 備考 |
|---------|--------|------|------|
| フェーズ1：要求分析 | 変更要求仕様書 | ✅完了／🔄作成中／⬜未着手 | |
| フェーズ2：スペックアウト | スペックアウト資料 | ... | |
| フェーズ3：設計 | 変更設計書 | ... | |
| フェーズ4：コーディング | ソースコード変更 | ... | |
| フェーズ5：テスト | テストケース／結果記録 | ... | |

### 現在のフェーズ
**フェーズN: {フェーズ名}**

### 次のアクション
→ `/xddp.XX.xxx` を実行してください
```

## 引数
$ARGUMENTS
