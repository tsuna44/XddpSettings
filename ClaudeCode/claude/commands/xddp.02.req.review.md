変更要求仕様書に対して、複数の視点でレビューを実施する。

## 実行内容

以下の4つの視点でレビューし、指摘事項を出力する：

### 1. 完全性チェック
- ユーザ要求→システム要求→仕様の階層が一貫しているか
- Before/After が全仕様に記載されているか
- 仕様番号の採番に漏れ・重複がないか

### 2. 明確性チェック
- 曖昧な表現（「適切に」「必要に応じて」）がないか
- 数値・条件が具体的に記述されているか

### 3. 一貫性チェック
- 用語が統一されているか
- 仕様間で矛盾がないか

### 4. テスト可能性チェック
- 仕様がテスト可能な形式で書かれているか
- 期待値・条件が検証可能か

## 出力形式
```
## レビュー結果

### 指摘事項一覧
| No | 仕様番号 | 視点 | 指摘内容 | 修正案 |
|----|---------|------|---------|--------|

### 総合評価
- 重大指摘：N件
- 軽微指摘：N件
- 次フェーズへの推奨：OK／要修正
```
