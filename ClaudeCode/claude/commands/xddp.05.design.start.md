変更要求仕様書・スペックアウト資料をもとに変更設計書を作成する。

## 実行内容

1. 変更要求仕様書とスペックアウト資料を読み込む
2. `templates/変更設計書-template.md` をベースに変更設計書を作成する
3. 各エントリに以下を記述する：
   - ヘッダ（仕様番号・ソースファイル・変更対象・変更種別）
   - 修正方針（設計の意図）
   - 関数外の変更（定数・型定義）
   - 関数内の変更（変更内容・予想行数）
   - 変更後の確認項目
4. 変更前コードを必ずソースから引用し、変更後との対比を明記する
