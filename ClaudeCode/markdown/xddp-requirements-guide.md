# xddp-requirements スキル 実行ガイド

## ファイル構成

```
your-project/
├── CLAUDE.md                        # プロジェクト共通ルール
├── .claude/
│   └── skills/
│       └── xddp-requirements/
│           └── SKILL.md             # 要求分析スキル
└── cr/
    ├── CR-2024-031-example.md       # 入力：変更要求書
    └── CR-2024-031-analysis.md      # 出力：要求分析結果（自動生成）
```

---

## Claude Code での実行方法

### 方法1：自然言語で指示（自動起動）
```
> cr/CR-2024-031-example.md のCRを要求分析して
```
→ Claudeがスキルの説明文を読み、xddp-requirementsを自動選択して実行

### 方法2：スラッシュコマンドで明示的に起動
```
> /xddp-requirements cr/CR-2024-031-example.md
```
→ スキルを直接指定して実行

---

## 実行の流れ（内部動作）

```
1. Claudeがシステムプロンプトでスキル一覧のメタデータを確認
        ↓
2. xddp-requirementsのdescriptionが合致すると判断
        ↓
3. SKILL.md の全文を読み込み（ここで手順・フォーマットを取得）
        ↓
4. CR-2024-031-example.md を読み込み
        ↓
5. Step1〜4を実行し、分析結果をMarkdownで出力
```

---

## 出力ファイルの活用

| 出力セクション | 次の工程での使い道 |
|--------------|-----------------|
| ウォンツ・ニーズ・制約 | 設計フェーズのインプット |
| ベースラインとの差分 | コード変更箇所の特定 |
| 影響範囲 | テスト計画のインプット |
| 懸念事項・確認事項 | 顧客・社内への確認アクション |
| TM更新指示 | トレーサビリティマトリクスの更新 |
