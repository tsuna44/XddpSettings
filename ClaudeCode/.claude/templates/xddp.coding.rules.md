# XDDP Step 07 コーディング品質ルール

> このファイルは `xddp.07.code` スキルが参照するコーディング品質ゲート定義。
> ルールのみを変更する場合はこのファイルだけを編集する。

---

## R1. エラーハンドリング実装ルール

| ルール | 内容 |
|--------|------|
| **例外の握りつぶし禁止** | 空の `catch` / `except` ブロックは禁止。例外は必ずログ記録するか上位に伝播させること |
| **リソースクリーンアップ** | ファイル・DB接続・ネットワーク接続は `finally` / `with` / `defer` 等で必ずクリーンアップすること |
| **タイムアウト設定** | 外部呼び出し（API・DB・ネットワーク）には必ずタイムアウトを設定すること |
| **エラー伝播の明示** | エラー発生時はログを記録してから呼び出し元に伝播させること（サイレント失敗禁止） |
| **フェイルセーフの実装** | CHD で設計されたフェイルセーフ方針（安全側への倒し方）が実装されていること |

---

## R2. 入力バリデーション・セキュリティルール

| ルール | 内容 |
|--------|------|
| **境界バリデーション** | 外部境界（ユーザー入力・外部APIレスポンス・ファイル読み込み）では必ず型・範囲・形式を検証すること |
| **機密情報ハードコード禁止** | APIキー・パスワード・トークン等を直接コードに記述してはならない（環境変数・設定ファイルを使うこと） |
| **インジェクション対策** | SQLは必ずプレースホルダを使用し、シェルコマンドへの外部入力埋め込みを禁止する |

---

## R3. ログ・観測可能性ルール

| ルール | 内容 |
|--------|------|
| **ログレベルの使い分け** | ERROR（障害・異常）/ WARNING（劣化・予期外）/ INFO（重要操作）/ DEBUG（詳細）を適切に分類すること |
| **エラーには文脈情報を含める** | エラーログにはスタックトレース・入力値・状態情報を含め、再現に必要な情報を残すこと |
| **デバッグ出力の残置禁止** | `print` / `console.log` 等の一時デバッグ出力をコミットに含めてはならない |

---

## R4. CHD 適合性ルール

| ルール | 内容 |
|--------|------|
| **設計との一致** | 実装は CHD の After コードと一致していること（独自判断での変更は設計差異として記録すること） |
| **確認項目の実装** | CHD の「確認項目」セクションに記載された動作が実装されていること |
| **スコープ外変更の禁止** | CHD に記載されていない変更を実装に含めてはならない |

---

## R5. レビューチェックポイント（品質ゲート）

静的検証・コードレビューでは以下を必ず確認すること：

### エラーハンドリング
- [ ] 空の `catch` / `except` ブロックがないか
- [ ] リソースリーク（未クローズのファイル・接続）がないか
- [ ] 外部呼び出しにタイムアウトが設定されているか
- [ ] エラー発生時に適切なログが記録されているか
- [ ] CHD で設計されたフェイルセーフ方針が実装されているか

### セキュリティ
- [ ] 外部境界で入力バリデーションが実施されているか
- [ ] 機密情報がハードコードされていないか
- [ ] SQLプレースホルダが使われているか（SQL操作がある場合）

### ログ・観測可能性
- [ ] エラーログに文脈情報（スタックトレース等）が含まれているか
- [ ] デバッグ用出力（print/console.log等）が残っていないか

### CHD 適合性
- [ ] 実装が CHD の After コードと一致しているか（エラー経路含む）
- [ ] CHD のスコープ外の変更が混入していないか
