# 変更要求書（CR）

## 基本情報
| 項目 | 内容 |
|------|------|
| CR番号 | CR-2024-031 |
| タイトル | 温度センサー異常検知のしきい値変更 |
| 起票日 | 2024-11-01 |
| 起票者 | 山田 太郎（品質保証部） |
| ステータス | 起票 |

---

## 1. 変更の背景・目的
現行システムでは温度センサーの異常検知しきい値が80.0℃に設定されているが、
設置環境の通常運転温度が75〜79℃の範囲に達することがあり、
誤検知アラームが頻発している。

その結果、保守担当者が不要な現地対応を月平均8件行っており、
対応コストと担当者の負荷が増大している。
しきい値を実態に合わせて見直すことで誤検知を削減し、
本来必要な異常のみを検知できる状態にしたい。

---

## 2. 顧客からの要求（ウォンツ）
「アラームが鳴りすぎて困っている。現場の作業員が毎回確認に行くのが負担になっている。
もう少し高めの温度で鳴るように設定を変えてほしい。
ただし、本当に危険な状態は絶対に見逃したくない。」

（2024-10-28 顧客定例会議にて 鈴木様（製造部長）より）

---

## 3. 制約条件
- 既存の警報ログフォーマット（CSV出力）は変更しない
- 他センサー（圧力・湿度）のしきい値ロジックには影響させない
- しきい値の上限は90.0℃とする（安全規格 IEC-XXXX 第4条に基づく）
- ソフトウェアバージョンは v2.3 系でリリースする（v3.0 系への移植は対象外）

---

## 4. 受入条件
- しきい値85.0℃に変更後、75〜84℃の範囲でアラームが発生しないこと
- 85.0℃以上でアラームが正常に発生すること
- 既存の回帰テスト（TC-040〜TC-050）が全てパスすること
- 警報ログのCSV出力フォーマットが変更前後で一致すること
