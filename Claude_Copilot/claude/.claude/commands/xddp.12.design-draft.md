# XDDP 変更設計書ドラフト生成

## 目的
スペックアウト資料で特定した変更箇所とTMの交点情報をもとに、
変更設計書のドラフトを生成する。
このドラフトはxddp.12.design-ai-reviewでAIレビューを受けた後、人間レビューへ進む。

## 入力
- TM（`TM-CR-YYYY-NNN.xlsx` または `TM-CR-YYYY-NNN.md`）← xddp.11.tm-generateで生成済み
  - ●の交点（仕様番号×ソースファイル）を変更エントリの単位として使用する
- スペックアウト資料（1ファイルまたは分割ファイル形式）
  - 変更要（●）の箇所・修正方針・処理構造・定数情報を参照する
- 変更要求仕様書（Before/Afterの参照用）

## 手順

### Step 1: TM交点の抽出
TMから●（変更あり）の全交点を抽出する：

```
（仕様番号）×（ソースファイル）→ 変更エントリ1件
```

複数の仕様番号が同一ファイルの同一関数に対応する場合は1エントリにまとめ、
ヘッダの仕様番号をカンマ区切りで列記する。

### Step 2: 各エントリのドラフト生成
スペックアウト資料を参照して各変更エントリを生成する：

**ヘッダ情報**
- 仕様番号：TM交点から取得
- ソースファイル：TM交点から取得
- 変更対象：スペックアウト資料の調査結果から特定（関数名・定数名等）
- ステータス：「作成中」固定
- 記述者・日付：省略（人間が記入）

**修正方針**
- スペックアウト資料の「変更方針」から転記する
- 記載がない場合は「（要記述：変更方針を記述してください）」と明示する

**データ構造の変更**
- スペックアウト資料の定数・構造体の変更要否テーブルから生成する
- 変更種別（ADD/MOD/DEL）・変更前後・理由を記述する
- 不明な場合は「（要確認）」と明示する

**関数外の変更**
- 定数・グローバル変数・型定義など関数の外の変更を記述する

**関数内の変更**
- スペックアウト資料の処理フローと変更方針から生成する
- 変更内容は文章で記述する（ソースコードは書かない）

**変更後の確認項目**
- 変更要求仕様書のBefore/Afterから境界値・条件分岐の確認項目を生成する
- 最低2件以上生成する（しきい値ちょうど・しきい値-1等）

### Step 3: 出力
以下のファイル名で出力する：
```
変更設計書-CR-YYYY-NNN.md
```

生成後に以下のサマリを表示する：
```
## 変更設計書ドラフト生成サマリ

| 項目 | 件数 |
|------|------|
| 変更エントリ数 | N件 |
| 要記述（修正方針未記入）| N件 |
| 要確認（不明項目あり） | N件 |

→ 次のステップ：/xddp.12.design-ai-review でAIレビューを実施してください
```

## 注意事項
- スペックアウト資料に「変更方針」が記載されていないエントリは
  必ず「（要記述）」と明示する。推測で方針を書かない
- 変更内容はソースコードではなく「何をどう変えるか」の文章で記述する
- TMに○（参照確認）のみで●がない仕様番号は変更エントリを生成しない
