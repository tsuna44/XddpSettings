# XDDP コーディング 指摘反映・AI再レビュー

## 目的
AIレビュー（xddp.17.coding-ai-review）または人間レビューの指摘を
ソースコードに反映し、修正後にAIが再レビューを実施する。
AI・人間どちらの指摘も同じコマンドで処理する。

## 入力
- 変更後ソースファイル（修正前）
- レビュー指摘（以下のいずれか）
  - AIレビューレポート（xddp.17.coding-ai-reviewの出力）
  - 人間レビューのコメント・指摘（テキストまたはファイル）
- 変更設計書（修正内容の根拠確認用）

## 手順

### Step 1: 指摘の種別判定
入力された指摘がAIレビューレポートか人間コメントかを判定する。
- AIレビューレポートの場合：🔴🟡🔵の分類に従って優先順位を付けて反映する
- 人間コメントの場合：全指摘を反映対象として扱う

### Step 2: 指摘の反映

修正前に以下を確認する：
- **設計書内の修正**（変更設計書の範囲内）→ そのまま修正する
- **設計書外の修正が必要**（変更設計書の記述が誤っている）→ 変更設計書の更新が必要。
  `/xddp.14.design-review-fix` で設計書を修正してから本コマンドを再実行する

| 対応種別 | 処理 |
|---------|------|
| 反映する | ソースコードを修正し変更タグを維持する |
| 一部反映 | 修正内容と理由を記録する |
| 反映しない | 却下理由を記録する（無断スキップ禁止） |
| 設計書修正が必要 | xddp.14.design-review-fixで設計書を先に修正する |

### Step 3: 修正サマリの出力

```markdown
## 指摘反映サマリ

**レビュー種別**：AIレビュー ／ 人間レビュー
**反映日**：YYYY-MM-DD

### 反映結果

| No | 指摘番号／内容 | 対応種別 | 修正内容 | 却下理由 |
|----|-------------|---------|---------|---------|
| 1 | 🔴-1（または人間指摘の内容） | 反映／一部反映／反映しない／設計書修正が必要 | （修正内容） | （却下の場合のみ） |
```

### Step 4: AIによる再レビューの実施
修正後のソースコードに対して、**必ずxddp.17と同じ別コンテキスト方式でAIが再レビューを実施する**。
修正箇所を中心に6観点でレビューし、新たなレビューレポートを出力する。

再レビュー後の出力に以下を追記する：

```
## 再レビュー後の次のアクション

### 🔴必須修正が残っている場合
→ /xddp.18.coding-review-fix を再実行して修正・再レビューを繰り返してください

### 🔴必須修正が0件になった場合
→ 人間レビューへ渡してください
  人間レビュー後：
  - 指摘がある場合 → 指摘内容を添えて /xddp.18.coding-review-fix を再実行
  - 承認の場合    → /xddp.19.verify で変更後確認へ進む
```

## ループの終了条件
AIレビューで🔴必須修正が0件になった後、人間レビューで承認が得られた時点でループを終了する。
**人間レビューは必ず実施すること。AIレビューのみで次フェーズへ進まない。**

## 注意事項
- 変更タグ（`[ADD/MOD/DEL-仕様番号]`）は修正後も必ず維持する
- 設計書外の修正が必要な場合は必ずxddp.14.design-review-fixに戻る
- 人間の指摘を優先する（AIと人間の指摘が混在する場合）
- ループ回数に制限はないが、3回を超える場合は変更設計書の見直しを推奨する
