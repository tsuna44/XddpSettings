# XDDP スペックアウト資料 指摘反映・AI再レビュー

## 目的
AIレビュー（xddp.08.specout-ai-review）または人間レビューの指摘を
スペックアウト資料に反映し、修正後にAIが再レビューを実施する。
AI・人間どちらの指摘も同じコマンドで処理する。

## 入力
- スペックアウト資料（修正前）
  - **1ファイル形式**：`スペックアウト-CR-YYYY-NNN.md` を修正して全文出力する
  - **分割ファイル形式**：指摘対象のファイルのみ修正する（他ファイルは再出力不要）
    - `specout-00-summary.md` は常に最新状態に更新する
- レビュー指摘（以下のいずれか）
  - AIレビューレポート（xddp.08.specout-ai-reviewの出力）
  - 人間レビューのコメント・指摘（テキストまたはファイル）

## 手順

### Step 1: 指摘の種別判定
入力された指摘がAIレビューレポートか人間コメントかを判定する。
- AIレビューレポートの場合：🔴🟡🔵の分類に従って優先順位を付けて反映する
- 人間コメントの場合：全指摘を反映対象として扱う

### Step 2: 指摘の反映
各指摘に対して以下の処理を行う：

| 対応種別 | 処理 |
|---------|------|
| 反映する | 1ファイル形式：該当箇所を修正 / 分割形式：対象ファイルを修正（summary.mdも更新） |
| 一部反映 | 修正内容と理由を記録する |
| 反映しない | 却下理由を記録する（無断スキップ禁止） |

### Step 3: 修正サマリの出力

```markdown
## 指摘反映サマリ

**レビュー種別**：AIレビュー ／ 人間レビュー
**反映日**：YYYY-MM-DD

### 反映結果

| No | 指摘番号／内容 | 対応種別 | 修正内容 | 却下理由 |
|----|-------------|---------|---------|---------|
| 1 | 🔴-1（または人間指摘の内容） | 反映／一部反映／反映しない | （修正内容） | （却下の場合のみ） |
```

### Step 4: AIによる再レビューの実施
修正後のスペックアウト資料に対して、**必ずAIが再レビューを実施する**。
xddp.08.specout-ai-reviewと同じ6観点でレビューを実施し、新たなレビューレポートを出力する。

再レビュー後の出力に以下を追記する：

```
## 再レビュー後の次のアクション

### 🔴必須修正が残っている場合
→ /xddp.09.specout-review-fix を再実行して修正・再レビューを繰り返してください

### 🔴必須修正が0件になった場合
→ 人間レビューへ渡してください
  人間レビュー後：
  - 指摘がある場合 → 指摘内容を添えて /xddp.09.specout-review-fix を再実行
  - 承認の場合    → /xddp.10.specout-feedback でフィードバックへ進む
```

## ループの終了条件
AIレビューで🔴必須修正が0件になった後、人間レビューで承認が得られた時点でループを終了する。
**人間レビューは必ず実施すること。AIレビューのみで次フェーズへ進まない。**

## 注意事項
- 指摘を「反映しない」と判断した場合は必ず却下理由を記録する
- AIと人間の指摘が混在する場合は、人間の指摘を優先する
- 修正対象のファイルのみ全文を出力する（差分のみの出力はしない）
- specout-00-summary.mdは常に最新状態を出力する
- ループ回数に制限はないが、3回を超える場合はスペックアウト調査自体の見直しを推奨する
