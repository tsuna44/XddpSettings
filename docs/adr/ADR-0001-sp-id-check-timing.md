# ADR-0001: SP-ID照合チェックを工程5開始時に行う理由

Status: Accepted
Date: 2026-07-11

## Context

工程4b（CRS 更新）で SP 項目が追加される場合があるため、工程4a完了時点で SP-ID 照合を行うと
工程4b更新分が反映されない。

## Decision

architect agent を呼び出す前（工程5開始時）に SP-ID 照合チェックを実行する。乖離が検出された場合は
処理を中断せず `WARNINGS` リストへ追加し継続する。

## Consequences

工程4b完了後の最新 CRS との乖離を検出できる。乖離時に処理を継続する設計は、CRS への SP 追加が
軽微な修正（定義補完）であることが多く、アーキテクトの判断で吸収できるという想定に基づく。
