# ADR-0002: カバレッジ自動補完でrepoが一意に決まらない場合に補完しない理由

Status: Accepted
Date: 2026-06-28

## Context

カバレッジ自動補完で、SP の担当 repo の有力候補が0個または2個以上（一意に決まらない）の場合、
自動補完をどう扱うかを決める必要がある。

## Decision

有力repoが「ちょうど1つ」の場合のみ自動補完する。0個または2個以上の場合、および補完後も
未解消の場合は自動補完を行わず `UNRESOLVED_MISSING` に記録し、人の判断に委ねる。

## Consequences

同一バッチ内に複数repo担当のSPが混在する場合に最初に見つかったrepoを機械的に採用すると、
誤ったリポジトリへの自動追記につながり、後続の工程7コーディングで誤実装が発生するリスクがある。
一意に決まらない場合を人の判断に委ねることでこのリスクを回避する。
