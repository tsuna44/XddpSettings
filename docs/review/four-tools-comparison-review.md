### 🟡 [6] Spec Workflow MCP の "4フェーズ" とセクション3の "5 MCP Tools" の説明不足（セクション1）

**該当箇所:**
- セクション1（行36）: `4フェーズ（Requirements → Design → Tasks → Implementation）`
- セクション3ヘッダ（行113）: `### Spec Workflow MCP v2.2.7（5 MCP Tools）`

**問題:**
ラウンド1から未修正。「フェーズ（ワークフロー段階）」と「MCPツール（API）」は異なる概念であるが、

### 🟡 [7] Implementation Notes "会話履歴に蓄積" の不正確な記述（セクション1・セクション4）

**該当箇所（2箇所）:**
- セクション1（行41）: `| 知見管理 | セッション内（Implementation Notes） | ...`
- セクション4（行147）: `| 知見管理 | Implementation Notes（会話履歴に蓄積） | ...`

**問題:**
ラウンド1から未修正（さらにセクション1にも同じ問題の別箇所が存在することを新規確認）。
「会話履歴に蓄積」「セッション内」という表現は、cc-sdd の Implementation Notes が一時的なメモリ（揮発的）であるかのような印象を与える。
cc-sdd の実際の動作は `tasks.md` の `## Implementation Notes` セクションに永続的に記録され、後続タスクへの知見伝播がファイルを通じて行われる。
「セッション内」という表現も「セッションが終われば消える」という誤解を招く。

**修正案:**
- セクション1（行41）: `セッション内（Implementation Notes）` → `tasks.md に記録（Implementation Notes）`
- セクション4（行147）: `Implementation Notes（会話履歴に蓄積）` → `Implementation Notes（tasks.md に記録・後続タスクに注入）`
