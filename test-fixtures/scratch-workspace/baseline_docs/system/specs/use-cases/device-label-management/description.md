## 2. 目的・ゴール

オペレーターがダッシュボードからデバイスに任意のラベル（例：「1F-倉庫」「critical」）を付与・削除できるようにする。1台のデバイスに複数ラベルを付与可能とし、設置場所や用途による分類を可能にすることで、異常時の対象特定を容易にする。

---

| アクター | 種別 | 説明 |

| Operator | 人（Operator） | ダッシュボードからデバイスにラベルを付与・削除する利用者（UR-001 に明示記述あり） |

---

## 4. 前提条件

- 対象デバイスが device-registry に登録済みであること
- オペレーターがダッシュボードにアクセス可能であること

---

## 5. 事後条件（成功時）

- 対象デバイスに指定したラベルが付与（または削除）されていること
- ラベル変更操作が監査ログに記録されていること（SP-009、audit-logger）

---

## 6. 主フロー概要

1. オペレーターがダッシュボードで対象デバイスを選択する
2. オペレーターがラベル（文字列）を入力し付与操作を実行する
3. device-registry がデバイスIDとラベル名を受け取り、デバイスのラベル配列に追加する
4. device-registry がラベル変更操作を監査ログに記録する（SP-009）
5. ダッシュボードに更新後のラベル一覧が反映される

---

## 7. 代替フロー

### 7a. ラベル削除

1. オペレーターがダッシュボードで既存ラベルの削除操作を選択する
2. device-registry がデバイスIDとラベル名を受け取り、デバイスのラベル配列から削除する
3. device-registry がラベル変更操作を監査ログに記録する（SP-009）
