### LL-003：リポジトリ間共有イベント型の breaking 変更はデプロイ順序の明記が必要

**CR：** CR-2026-900 ／ **工程：** 方式検討 ／ **repo：** cross ／ **タグ：** #設計, #方式検討

**発生状況：**
`device.alert.raised` イベント（共有データ型 `AlertEvent`）に必須フィールド `labels` を追加した際、device-svc と notify-svc を同時にデプロイしないと旧形式イベントの処理に問題が生じる可能性があることに気づいた。

**学んだこと：**
複数リポジトリで共有するイベント・データ型に breaking 変更を加える場合、CHD のインタフェース変更サマリに breaking フラグを立てるだけでなく、デプロイ順序・移行手順まで明記しないと運用時に問題が起こり得る。

**次回への適用：**
- cross の CHD で breaking: true とした項目は、デプロイ順序または後方互換措置（フィールドのoptional化等）を必ず記載する
