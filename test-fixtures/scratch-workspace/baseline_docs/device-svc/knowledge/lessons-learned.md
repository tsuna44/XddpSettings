### LL-001：sensor_reader.py の閾値判定は境界値に弱い

**CR：** CR-2026-900 ／ **工程：** テスト ／ **repo：** device-svc ／ **タグ：** #リスク

**発生状況：**
テスト実行（TC-101）で、センサー値が THRESHOLD_HIGH と同値のときアラートが発火しない不具合（NG-001）を検出した。

**学んだこと：**
`src/sensor_reader.py` の `read_sensor()` は閾値比較に `>` を使っており、境界値（同値）を含めない実装になっていた。同様のパターンが他の閾値判定にも潜んでいる可能性がある。

**次回への適用：**
- 閾値判定を追加・変更する際は `>` と `>=` のどちらが要求に合致するかを必ず確認する
- sensor_reader.py 内の他の閾値判定箇所も棚卸しを推奨
